FAQ

Q: What is an "instance"?
A: An instance is a virtual server running on either a public or private cloud network. Instances provide places for application code to live in the cloud, and enable applications to easily scale up and down with traffic by creating and destroying instances on the fly.

Q: What are some use cases for this tool?
A: There are many ways CloudMaker could be used in the future. Application developers could use CloudMaker to quickly create environments for demoing prototype applications. QA/Testers could use CloudMaker to rapidly generate many instances and run automated tests on them, reducing testing time.

Q: What kinds of instances are created by this tool?
A: Currently, CloudMaker only creates AWS (Amazon Web Services) EC2 t2.micro instances. Each of these instances comes with 1 virtual CPU and 1GiB of memory included. You can read more about EC2 instances here. CloudMaker could easily be updated in the future to create more types of instances and allow more detailed configuration of the instances.
